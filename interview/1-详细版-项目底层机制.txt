一、详细版：项目底层机制（全链路）

技术栈
- Spring Boot 3.2.5 + Java 17
- JPA / Hibernate（操作数据库）
- H2 数据库（内嵌，不需要安装）
- Thymeleaf（模板引擎，后端生成 HTML）
- Maven（构建工具）

==============================================================================
从注册到下单，后端每一步
==============================================================================

1. 注册

用户填表单 -> 浏览器 POST 到 /register
-> UserController 接收请求
-> UserService 检查用户名是否已存在
-> 不存在则 UserRepository.save(user) 写入数据库
-> 跳转到登录页

知识点：
- 三层架构：Controller（接收请求）-> Service（业务逻辑）-> Repository（操作数据库）
- JPA：不用写 SQL，save() 方法自动 INSERT INTO

2. 登录

用户填账号密码 -> POST /login
-> 查数据库验证密码
-> 正确的话，session.setAttribute("userId", id)
-> 服务端把 sessionId 通过 Cookie 返回浏览器
-> 后续请求浏览器自动带上 Cookie，服务端就知道是谁

知识点：
- HTTP 是无状态的，Session + Cookie 让服务器"记住"用户
- 每次请求从 session.getAttribute("userId") 拿当前用户

3. 浏览商品

用户访问首页 -> GET /
-> PageController.index()
-> GoodsService.getAllGoods() -> 查数据库拿所有商品
-> 数据塞进 Model
-> 渲染 index.html（Thymeleaf 模板）
-> 返回 HTML 页面给浏览器

知识点：
- MVC：Model（数据）-> View（模板）-> Controller（调度）
- Thymeleaf 在服务端把商品列表渲染成 HTML，浏览器直接展示

4. 下单（核心流程）

用户点"立即购买" -> 弹窗显示微信收款码 -> 点"已支付"
-> POST /order/create（携带 goodsId、count）
-> OrderController.createOrder()
-> OrderService：
   ① 查数据库拿到商品信息（价格、库存）
   ② 计算总价 = 价格 x 数量
   ③ 扣库存：原库存 - 购买数量
   ④ 创建订单：保存到 orders 表
-> 重定向到 /orders 页面查看订单

知识点：
- 下单涉及"扣库存"和"建订单"两步操作
- 两步必须同时成功或同时失败

5. 查看订单

用户访问"我的订单" -> GET /orders
-> 从 session 拿到 userId
-> OrderService.findByUserId(userId) 查数据库
-> 拿到该用户所有订单，渲染页面

==============================================================================
AI 助手模块（四层架构）
==============================================================================

第一层：记忆层
- 每个用户有一个独立的聊天记录列表
- 用 HashMap 存：userId -> 消息列表
- 限制最多 20 条，超过就删最早那条
- 作用：让 AI 记得之前聊过什么，实现连续对话

第二层：规划层（核心）
- 配了 DeepSeek API Key 就用大模型理解语义
- 没配就用关键词匹配兜底
- 大模型模式：先把数据库里的商品和订单查出来，塞进"系统提示词"，
  再让 DeepSeek 基于真实数据回复
- 这就是 RAG（检索增强生成）——从数据库检索数据 + 让 AI 生成回复

第三层：工具层
- 每个功能是一个工具函数：
  - 查订单、查商品、推荐商品、价格筛选、联系客服等
- 每个工具都去查数据库，返回真实数据

第四层：生成层
- 返回两样东西：自然语言回复（显示在聊天框）+ 结构化数据（前端渲染商品卡片）
- 前端收到数据后，聊天框显示文字，下方展示商品卡片

==============================================================================
数据库相关
==============================================================================

当前：H2 数据库
- 内嵌在 Java 进程里，不需要安装
- 以文件形式存在硬盘上（data/shopdb.mv.db）
- 适合开发和演示

生产：可以切换 MySQL
- 已预留 MySQL 配置（pom.xml 有驱动）
- 只需要设环境变量 SPRING_PROFILES_ACTIVE=mysql 和 MYSQL 连接信息
- 代码一行不用改，因为 JPA 屏蔽了数据库差异

==============================================================================
几个基础概念解释
==============================================================================

三层架构：Controller 管接收请求，Service 管业务逻辑，Repository 管数据库。
下层对上层说"你要什么数据我给你"，上层对下层说"我要什么你帮我查"。

JPA：Java 自带的数据库操作标准。写一个 Entity 类（比如 Goods），JPA 自动生成
对应的表。写 repository.save() 就是 INSERT，findAll() 就是 SELECT *。

Session + Cookie：用户登录后，服务端在内存里记下"userId=1"，返回一个 sessionId
给浏览器存成 Cookie。后面每次请求，浏览器带着这个 Cookie 来，服务端就知道是谁了。

MVC：Model（数据）-> View（页面模板）-> Controller（调度）。Controller 先查数据
放 Model 里，再把 Model 交给 View 渲染成 HTML。
