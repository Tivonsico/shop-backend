## 项目介绍正文

"我做的是一个电商网站，用 Spring Boot 写的后端。功能包括注册登录、商品浏览、下单购买，还有一个 AI 客服聊天。

后端分了三层——Controller 管接收请求、Service 管业务逻辑、Repository 管数据库。用户下单时，先用 session 判断有没有登录，登录了才允许下单。下单的逻辑很简单，查商品价格、算总价、扣库存、建订单。

数据库用的 H2，是内嵌的不用安装，启动项目自动就跑了。通过 JPA 操作数据库，用 Java 对象直接 save 和 find，不需要写 SQL。

AI 聊天模块用了四层架构：记忆层存聊天记录、规划层接入了 DeepSeek 的 API 做语义理解、工具层去数据库查商品和订单的真实数据、生成层把结果返回前端展示。这个方案其实就是 RAG——先检索真实数据，再让大模型回答。

项目是借助 Claude 完成的，但我很清楚每块代码的原理。比如 session 怎么保持登录态、JPA 怎么映射数据表、下单的事务怎么保证数据一致性，我都能说清楚。"

---

## 如果面试官问"用了什么技术"

"Spring Boot 做后端框架，JPA 操作数据库，H2 做开发数据库，Thymeleaf 做页面模板，DeepSeek API 做 AI 对话。部署在 Railway 上，代码放 GitHub 自动部署。"

---

## 如果面试官问"遇到什么困难"

"有两个印象比较深的。一是部署上线时数据库配置不对，应用一直崩溃，排查后发现是环境变量冲突，清理掉就好。二是 AI 模块接 DeepSeek API 的时候，大模型回复格式不符合预期，我改成了先查数据库再让大模型自由对话，效果就好了。"

## 核心要点（不要忘）

✅ 项目是啥：一个电商网站，有注册登录、商品展示、下单、AI 聊天
✅ 技术栈：Spring Boot + JPA + H2 + Thymeleaf
✅ 三层架构：Controller → Service → Repository
✅ 下单做了什么：查商品 → 算价格 → 扣库存 → 建订单
✅ AI 四层架构：记忆层 → 规划层 → 工具层 → 生成层
✅ 项目借助 Claude 完成，但原理都懂
