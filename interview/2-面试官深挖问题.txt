二、面试官可能深挖的问题和答案

==============================================================================
Q1：为什么要用三层架构（Controller/Service/Repository）？
==============================================================================

答：让代码职责清晰。Controller 只管接收 HTTP 请求和返回结果，Service 只管写
业务逻辑，Repository 只管操作数据库。这样改了数据库实现，上层代码不用动；
改了业务逻辑，Controller 也不用动。职责单一，好维护。

==============================================================================
Q2：@Transactional 是干什么的？
==============================================================================

答：保证一个方法里的多个数据库操作要么全部成功、要么全部失败。比如下单时先扣
库存再建订单，如果扣库存成功了但建订单失败了，加了 @Transactional 会自动回滚，
把库存恢复回去，不会出现"库存扣了但订单没创建"的情况。

==============================================================================
Q3：Session 和 Cookie 的区别？
==============================================================================

答：Session 存在服务端，Cookie 存在浏览器。用户登录后，服务端创建一个 Session
存储用户信息，同时把 sessionId 通过 Cookie 发给浏览器。后续请求浏览器自动带上
Cookie，服务端根据 sessionId 找到对应的 Session，就知道是谁在访问了。

==============================================================================
Q4：为什么用 JPA 不用 SQL？
==============================================================================

答：JPA 让我用 Java 对象的方式操作数据库，不用写 SQL。比如 UserRepository.save(user)
就是插入一条用户数据，findById(1) 就是 SELECT * FROM user WHERE id=1。而且
更换数据库（比如从 H2 换成 MySQL），代码不需要改。

==============================================================================
Q5：H2 和 MySQL 有什么区别？
==============================================================================

答：H2 是内嵌数据库，不需要安装，启动项目时自动运行，存在一个文件里。MySQL 是
一个独立的数据服务，需要安装配置。开发阶段用 H2 方便，生产环境换成 MySQL 性能
更好、支持并发。我的项目已经预留了 MySQL 配置，切换只需要改一个配置文件和设置
环境变量。

==============================================================================
Q6：AI 助手记忆层为什么用 HashMap？
==============================================================================

答：HashMap 是最简单的键值对存储，key 是用户 ID，value 是聊天记录列表。项目
初期验证功能用 HashMap 够了。如果上线用户多了，可以换成 Redis 存到内存里，
或者存到 MySQL 表里持久化。

==============================================================================
Q7：什么是 RAG？
==============================================================================

答：RAG 是"检索增强生成"。我的 AI 在回答用户问题前，先查数据库拿到真实的商品
和订单数据，再把数据塞给 DeepSeek 大模型，让大模型基于这些真实数据回答。这样
AI 就不会自己瞎编，回答的内容都是真实的。

==============================================================================
Q8：Spring Boot 和 Spring 的区别？
==============================================================================

答：Spring Boot 是 Spring 的增强版，帮我们做了很多自动配置。比如我想用数据库，
Spring Boot 会自动配置好数据源和 JPA；想搭 Web 服务，自动配好 Tomcat。不需要
写一堆 XML 配置文件，加个依赖、写几行配置就能跑起来。

==============================================================================
Q9：@Autowired 是干什么的？
==============================================================================

答：依赖注入。就是告诉 Spring："我这个类需要用到另一个类的实例，你帮我创建好
传给我"。比如 Controller 需要 Service，加 @Autowired 或者写在构造函数里，
Spring 会自动把 Service 实例传进去，不用我们手动 new。

==============================================================================
Q10：项目遇到过什么困难？怎么解决的？
==============================================================================

（挑一个说）

选 AI 模块那个：
"最麻烦的是接入 DeepSeek API 后，AI 的回复总是匹配不上。我排查后发现大模型
返回的是自然语言，但代码里强行让它输出固定格式，结果它不听话。我重新设计了
方案——先把数据库里的商品数据查出来放提示词里，让大模型直接自由对话，不再
限制格式，问题就解决了。"

或者选数据库那个（更基础）：
"部署到线上后，应用总是崩溃，报错说连不上数据库。排查发现是因为环境变量还配着
MySQL 的配置，但线上只有 H2。我删掉了 MySQL 配置文件，加上了 spring.jpa.
database-platform=H2Dialect 强制指定数据库方言，问题解决。"

==============================================================================
Q11：Java 和 C++ 有什么区别？
==============================================================================

（你 C++ 转 Java，这个几乎必问！）

答：① Java 有自动垃圾回收，不用手动 delete/free 内存；② Java 跨平台，编译成
字节码在 JVM 上跑，Windows/Mac/Linux 都能跑；③ Java 没有指针，有更安全的内存
管理；④ Java 的异常是 try-catch 机制，比 C++ 更规范；⑤ Java 生态更丰富，
Spring 全家桶开箱即用。但 C++ 性能更高、更底层，学 C++ 让我更能理解内存和指针
的原理。

==============================================================================
Q12：你作为大二学生，这个项目最大的收获是什么？
==============================================================================

答：最大的收获是完整地走了一遍后端开发的流程——从搭项目、写接口、连数据库、
部署上线到排查问题。以前只会写 C++ 算法题，做完这个项目才知道一个网站是怎么
跑起来的：前端请求经过 Controller -> Service -> 数据库，数据再一层层返回渲染
成页面。也学会了遇到问题怎么查日志、怎么定位 bug，这些是刷题学不到的。
